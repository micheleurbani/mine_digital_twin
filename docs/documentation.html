Documentation of Mine.py¶

In the following, the documentation of the classes in the module Mine.py is produced automatically. Mine.py contains all the classes for the objects currently in use in the simulation and the simulation procedures as presented in Get Started. At the bottom of the page, the documentation of some Simpy resources classes is reported.

class Mine.DumpSite(env, id, coordinates, mu, sigma)¶

The class inherits the basic attributes from Server and replicates the behavior of a dumpsite.

class Mine.Server(env, id, coordinates, mu=None, sigma=None)¶

This is the base class with the common attributes and methods of all server entities within the simulation.

class Mine.Shovel(env, id, coordinates, mu, sigma, alpha, beta, muPreventive, sigmaPreventive, muCorrective, sigmaCorrective, Cc, Cp, p, workshops)¶

The class inherits the basic attributes from Server. It includes the method to replicate the operations in case of preventive maintenance of a shovel.

Parameters

  • env (obj) – A simpy.Environment object

  • id (int) – Identification number

  • coordinates (tuple) – the (x, y) coordinates of the site on the map

  • mu (float) – mean of the (lognormal) serving time distribution

  • sigma (float) – st. dev. of the (lognormal) serving time distribution

  • alpha (float) – Scale parameter of Weibull distribution

  • beta (float) – Shape parameter of Weibull distribution

  • muCorrective (float) – Mean time to repair

  • sigmaCorrective (float) – Time to repair std. dev.

  • muPreventive (float) – Mean maintenance time

  • sigmaPreventive (float) – Maintenance time std. dev.

  • Cc (float) – Cost of corrective maintenance

  • Cp (float) – Cost of preventive maintenance

  • p (float) – Probability threshold for preventive maintenance

  • workshops (list) – The list of workshops in the system

doPreventiveMaintenance(expTaskTime)¶

The decision rule to trigger a preventive maintenance intervention is enclosed in the method. The rule used is the following.

\[P(t < T < \Delta t | T > t) > \ ext{threshold}\]
preventiveMaintenance(workshop)¶

The method is a generator function which replicates the maintenance operations for a shovel in case of preventive maintenance.

class Mine.Truck(alpha, beta, muCorrective, sigmaCorrective, muPreventive, sigmaPreventive, Cc, Cp, p, env, id, shovels, dumpsites, workshops, muCapacity, sigmaCapacity)¶

The class replicates the operations and maintenance process of a Truck. The following parameters must be passed to initialize the object.

Parameters

  • alpha (float) – Scale parameter of Weibull distribution

  • beta (float) – Shape parameter of Weibull distribution

  • muCorrective (float) – Mean time to repair

  • sigmaCorrective (float) – Time to repair std. dev.

  • muPreventive (float) – Mean maintenance time

  • sigmaPreventive (float) – Maintenance time std. dev.

  • Cc (float) – Cost of corrective maintenance

  • Cp (float) – Cost of preventive maintenance

  • p (float) – Probability threshold for preventive maintenance

  • env (obj) – A simpy.Environment object

  • id (int) – Identification number

  • shovels (list) – The list of shovels in the system

  • dumpsites (list) – The list of dump sites in the system

  • workshops (list) – The list of workshops in the system

assignment(items)¶

The function returns the item \(i\) (shovel, dumpsite, workshop) to which the Truck is assigned among a set of items \(I\), which is provided as a list. The value \(i\) is returned by the following equation.

\[\ ext{argmin}_{i \in I} = \mathbb{E} [ \ ext{travel distance to $i$} ] + \mathbb{E} [ \ ext{waiting time at $i$} ]\]
doPreventiveMaintenance(expTaskTime)¶

The decision rule to trigger a preventive maintenance intervention is enclosed in the method. The rule used is the following.

\[P(t < T < \Delta t | T > t) > \ ext{threshold}\]
fault()¶

The method generates a fault every now and then.

run(shovels, dumpsites, workshops)¶

The method is a generator function which returns the event linked with the operations of a truck.

class Mine.WorkShop(env, id, coordinates)¶

The class inherits the basic attributes from Server and replicates the behavior of a workshop. The serving time depends on the repair time of the item under repair and on the type of maintenance, i.e., corrective or preventive.

Simpy resources¶

class simpy.Resource(env, capacity=1)¶

Resource with capacity of usage slots that can be requested by processes.

If all slots are taken, requests are enqueued. Once a usage request is released, a pending request will be triggered.

The env parameter is the Environment instance the resource is bound to.

property count¶

Number of users currently using the resource.

queue = None¶

Queue of pending Request events. Alias of put_queue.

release¶

Release a usage slot.

alias of Release

request¶

Request a usage slot.

alias of Request

users = None¶

List of Request events for the processes that are currently using the resource.

class simpy.PriorityResource(env, capacity=1)¶

A Resource supporting prioritized requests.

Pending requests in the queue are sorted in ascending order by their priority (that means lower values are more important).
